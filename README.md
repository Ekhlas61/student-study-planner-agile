#  Student Study Planner App  
## Agile Project Simulation using Scrum, Jira, and GitHub  

---

## Overview  
The **Student Study Planner App** is a simulated software project designed to demonstrate the practical application of **Agile methodologies**, specifically the **Scrum framework**, in a collaborative development environment.

Rather than focusing on building a production-ready system, this project emphasizes:
- Structured Agile workflows  
- Team collaboration  
- Iterative development  
- Use of industry-standard tools  

The application allows students to:
- Create and manage study tasks  
- Set deadlines  
- Track progress  
- Organize their study schedule  

---

## Project Objectives  
- Simulate a real-world Agile software development lifecycle  
- Apply Scrum practices (Sprints, Standups, Retrospectives)  
- Manage tasks and backlog using Jira  
- Implement version control and collaboration using GitHub  
- Demonstrate team coordination and workflow management  

---

## Agile Framework  

### Scrum Implementation  
The project follows a **two-Sprint structure**, each lasting two weeks:

- **Sprint 1:** Core setup and basic functionality  
- **Sprint 2:** Feature completion and system refinement  

###  Key Agile Practices  
- Product Backlog management  
- Sprint Planning  
- Daily Standups  
- Sprint Reviews  
- Sprint Retrospectives

## Branching Strategy  

- **main** → Stable release version  
- **develop** → Integration branch  
- **feature/*** → Feature-specific branches  

### Example branches:
- feature/login  
- feature/register  
- feature/create-task  
- feature/deadline  
- feature/progress  

---

## Commit Strategy  

Commits are clear and linked to Jira user stories:

Added login UI (US-102)
Implemented task creation (US-201)
Added deadline feature (US-301)


This ensures:
- Traceability  
- Transparency  
- Easy progress tracking  

---

## Pull Request Workflow  

1. Create a feature branch from `develop`  
2. Implement changes  
3. Commit and push updates  
4. Open a Pull Request  
5. Review and merge into `develop`  

---

## Release Management  

- **v1.0** → Core features (authentication, task creation)  
- **v1.1** → Additional features (deadlines, progress tracking, notifications)
